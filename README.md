These scripts are intended to merge all the Chimera output files into one hdf5, which is averaged over equivalent momenta, over irrep rows and correcting the signs of the baryons going backwards and crossing the temporal boundary. 
* transforming_to_hdf5.py: This is the main script to run. This file creates the hdf5 file needed for the CLSAnalysis scripts. It requires the directories where the data will be saved and the type of source time to transform (t00_fwd, t00_bwd, etc.). For the Single hadrons case, the dat files from Chimera must be in the same folder and have a name that containes the string of the source time. 
* streams_dic.py: contains the informatino of the gauge configs, the names of the outut hdf5 files and some tags about going forward or backward to correct for the signs. The name of the txt file with the source time positions must be specified here too, the full path is recommended.
* hadron_data.py: It contains the information about the hadrons to merge. In the Single Hadron cases it differentiates between meson and baryons so it includes the sign change when crossing the temporal boundary. The operators must be defined here too, along with the names of the irreps and the momentum.
* merging_hdf5.py: Merges all the hdf5s created for each source time, i.e., t00_fwd, t00_bwd, t01_fwd and t01_bwd. It averages over source times if it finds repeated configs, otherwise it just puts them all together sorted by gauge configuration number,
